## A Batalha dos Code Agents

Nos últimos 6 meses, usei tanto GitHub Copilot quanto Cursor intensivamente em projetos reais. Neste artigo, vou compartilhar minha experiência honesta com ambos, incluindo métricas reais de produtividade.

**Spoiler**: Não há um "vencedor" absoluto. Cada ferramenta brilha em contextos diferentes.

## Minha Metodologia de Teste

Testei ambas as ferramentas em:
- **Projeto 1**: API Rest em Node.js (15.000 linhas)
- **Projeto 2**: Frontend React (23.000 linhas)
- **Projeto 3**: Scripts Python para ML (8.000 linhas)
- **Projeto 4**: Refatoração de código legado (50.000 linhas)

**Métricas coletadas**:
- Tempo economizado
- Qualidade do código sugerido
- Taxa de aceitação de sugestões
- Número de bugs introduzidos
- Custo mensal

## GitHub Copilot: Visão Geral

### Pontos Fortes

**1. Integração Perfeita**
- Funciona nativamente no VS Code
- Setup em literalmente 2 cliques
- Zero configuração necessária

**2. Autocomplete Excepcional**
```javascript
// Apenas digitei o comentário:
// função para validar email

// Copilot gerou:
function validarEmail(email) {
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return regex.test(email);
}
```

**Taxa de aceitação**: 68% das sugestões

**3. Ótimo para Código Boilerplate**

Tarefas repetitivas onde Copilot brilha:
- CRUD operations
- Testes unitários
- Configs (webpack, tsconfig, etc)
- Tipos TypeScript

**Exemplo real**:
```typescript
// Escrevi:
interface User

// Copilot completou:
interface User {
  id: string;
  name: string;
  email: string;
  createdAt: Date;
  updatedAt: Date;
  role: 'admin' | 'user' | 'guest';
}
```

### Limitações

**1. Contexto Limitado**

Copilot vê apenas o arquivo atual. Em projetos grandes, isso é problemático:

```javascript
// arquivo utils/database.ts
export class DatabaseService { ... }

// arquivo api/users.ts
// Copilot não sabe que DatabaseService existe
// Sugestões são genéricas demais
```

**2. Não Entende Arquitetura do Projeto**

Sugere padrões inconsistentes com o resto do código.

**3. Chat Limitado**

Copilot Chat existe, mas é básico comparado ao Cursor.

### Métricas Reais (3 meses)

- **Tempo economizado**: 4-6 horas/semana
- **Linhas aceitas**: 2.840
- **Taxa de aceitação**: 68%
- **Bugs introduzidos**: 3
- **Custo**: $10/mês (individual) ou $19/mês (business)

## Cursor: Visão Geral

### Pontos Fortes

**1. Contexto do Projeto Completo**

Cursor indexa TODO o projeto:

```
// Posso perguntar:
"Onde está a lógica de autenticação?"

// Cursor responde com arquivos relevantes:
- src/auth/middleware.ts (linha 45)
- src/api/login.ts (linha 12)
- src/utils/jwt.ts (linha 78)
```

**Isso é GAME CHANGER**.

**2. Chat Inteligente**

Chat do Cursor entende contexto multi-arquivo:

```
Eu: "Refatore a função getUserData para usar async/await"

Cursor: 
1. Encontra getUserData em 3 arquivos diferentes
2. Mostra preview das mudanças
3. Aplica mudanças em todos os arquivos
4. Atualiza testes relacionados
```

**3. Modo Compose** (Killer Feature)

```
Prompt: "Crie endpoint POST /api/users que:
- Valide email
- Hash password com bcrypt
- Salve no banco
- Retorne JWT
- Adicione testes"

Cursor:
✓ Cria arquivo routes/users.ts
✓ Atualiza database/models.ts
✓ Cria tests/users.test.ts
✓ Atualiza documentação API
```

**Produtividade**: Economizei 2 horas nessa tarefa.

**4. Multi-File Edit**

```
Prompt: "Renomeie 'userId' para 'user_id' em todo projeto"

Cursor:
- Encontra 47 ocorrências em 12 arquivos
- Mostra preview
- Aplica mudanças atomicamente
```

Isso é INSANO de útil.

### Limitações

**1. Custo Maior**

- $20/mês (Pro plan)
- Requests limitados no plano free

**2. Curva de Aprendizado**

Copilot é plug-and-play. Cursor requer:
- Entender como usar @ references
- Aprender comandos (Cmd+K, Cmd+L)
- Configurar .cursorrules

**3. Performance**

Em projetos muito grandes (&gt;100k linhas):
- Indexação inicial lenta (~5 min)
- Algumas queries demoram

**4. Precisa de Bons Prompts**

Garbage in, garbage out. Você precisa ser específico:

❌ Ruim: "Melhore o código"
✅ Bom: "Refatore usando strategy pattern, adicione error handling e escreva testes"

### Métricas Reais (3 meses)

- **Tempo economizado**: 8-12 horas/semana
- **Features completadas**: 34
- **Taxa de sucesso**: 82%
- **Bugs introduzidos**: 5
- **Custo**: $20/mês

## Casos de Uso: Qual Usar?

### Use GitHub Copilot Quando:

1. **Projeto pequeno/médio** (&lt;10k linhas)
2. **Trabalho focado em um arquivo**
3. **Código boilerplate/repetitivo**
4. **Budget limitado**
5. **Quer algo simples e direto**

**Perfil ideal**: Desenvolvedor que quer autocomplete inteligente sem complexidade.

### Use Cursor Quando:

1. **Projeto grande** (&gt;10k linhas)
2. **Refatoração em múltiplos arquivos**
3. **Arquitetura complexa**
4. **Precisa entender codebase desconhecida**
5. **Quer IA como pair programmer**

**Perfil ideal**: Desenvolvedor que quer maximizar produtividade e está disposto a investir tempo aprendendo.

## Minha Recomendação

**Para iniciantes**: Comece com GitHub Copilot
- Mais fácil
- Mais barato
- Menor curva de aprendizado

**Para profissionais**: Vale a pena Cursor
- ROI justifica o custo
- Produtividade muito maior
- Features avançadas valem o esforço

**Solução ideal**: Use ambos!

Eu uso:
- **Cursor**: Como IDE principal
- **Copilot**: Em outros editors (Vim, etc)

## Dicas Para Maximizar Produtividade

### Com Copilot:

1. **Escreva comentários descritivos**
```javascript
// Função que valida CPF brasileiro e retorna true se válido
// Remove pontos e traços, verifica dígitos verificadores
```

2. **Use nomes descritivos**
```javascript
// ❌
function validate(x)

// ✅
function validateBrazilianCPF(cpf: string)
```

3. **Aceite parcialmente**
- Use `Tab` para aceitar
- Use `Alt+]` para próxima sugestão
- Use `Alt+[` para anterior

### Com Cursor:

1. **Use @ para contexto**
```
@arquivo.ts refatore usando async/await
```

2. **Seja específico**
```
❌ "Adicione validação"
✅ "Adicione validação Zod para campos email, password (min 8 chars) e name"
```

3. **Use .cursorrules**
```
// .cursorrules
- Use TypeScript strict mode
- Prefira async/await a promises
- Sempre adicione error handling
- Escreva testes para funções públicas
```

Ambas as ferramentas são excelentes. A escolha depende de:
